# Convert admin product editor to task tabs and add slug SEO autofill

## Goal

Make the admin product editor faster for data entry by reducing always-visible form length and adding local smart-fill actions for repetitive slug/SEO fields, without changing schemas, server actions, publish checks, or saved payload shape.

## Scope

- `src/components/admin/catalog/product-form.tsx`

## Tasks

1. Replace the long always-visible main column with task tabs:
   - `Setup`
   - `Content`
   - `Pricing`
   - `Taxonomy`
   - `Publish`

2. Add smart-fill controls for each locale:
   - Generate slug from title.
   - Copy title to SEO title.
   - Copy description summary to SEO description.

3. Keep the existing sidebar actions and readiness.
   - Save/publish behavior remains unchanged.
   - Workflow links remain visible after draft creation.

4. Verify.
   - Run `npm run typecheck`.
   - Run `npm run lint`.

## Done Criteria

- Form no longer displays all major sections at once.
- Repetitive slug/SEO entry is faster through local helper buttons.
- Existing submit and publish behavior is unchanged.
