# Storefront Public Loaders Operational Instrumentation

## Goal

Record sanitized operational failures for public storefront loaders so catalog, blog, and review data failures are visible to admins/developers without exposing internal database details to visitors.

## Scope

- Public catalog RPC failures for list, facets, product detail, category detail, and collection detail.
- Public blog list/detail RPC failures.
- Public product review query failures.
- Keep raw database errors, private relation names, review body text, emails, tokens, and payloads out of operational facts.

## Plan

1. Add failing tests for operational recorder calls on catalog, blog, and public review load failures.
2. Instrument the loaders with `recordOperationalFailure` before returning/throwing their existing safe errors.
3. Run targeted unit tests, typecheck, lint, then update GSD summary/state and commit only scoped files.

## Verification

- `npm run test:unit -- tests/unit/catalog/queries.test.ts tests/unit/content/blog.test.ts tests/unit/reviews/eligibility.test.ts`
- `npm run typecheck`
- `npm run lint`
