# 260708-ops20 - Checkout Exception Operational Instrumentation

## Goal

Record sanitized operational failures for checkout market exception request, approval, rejection, and grant validation flows so customer/admin failures are diagnosable without exposing customer email, notes, grant tokens, token hashes, or raw database details.

## Scope

- Add focused unit coverage for checkout exception failure recording.
- Instrument `src/checkout/exceptions.ts` error branches with `recordOperationalFailure`.
- Keep invalid input, invalid request state, and expired/invalid grants as non-operational outcomes.
- Preserve existing result contracts and admin authorization ordering.
