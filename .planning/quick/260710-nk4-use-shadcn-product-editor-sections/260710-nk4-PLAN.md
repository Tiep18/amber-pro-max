# Use shadcn components for product editor section controls

## Goal

Polish the admin product editor smart sections with existing shadcn-style project components instead of hand-rolled button/card styling.

## Scope

- `src/components/admin/catalog/product-form.tsx`

## Tasks

1. Refactor section headers to compose existing UI primitives.
   - Use `Card`, `CardHeader`, `CardTitle`, `CardContent`, `Button`, and `Separator`.
   - Keep section toggles available in the form flow.
   - Improve spacing, hierarchy, and affordance without changing behavior.

2. Preserve editor logic.
   - Keep `openSections`, save/publish behavior, server actions, schemas, SEO, ISR, and storefront code unchanged.

3. Verify.
   - Run `npm run typecheck`.
   - Run `npm run lint`.

## Done Criteria

- Section controls look consistent with existing shadcn-style components.
- The UX remains smarter while scrolling.
- Typecheck and lint pass.
