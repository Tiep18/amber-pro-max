# Redesign admin product editor as a scrollspy workspace

## Goal

Turn the long admin product form into a stable two-column editing workspace: all form sections remain in document flow, while a viewport-height control rail provides status, section navigation, and persistent actions.

## Locked Decisions

- Preserve catalog server actions, schemas, payloads, cache invalidation, SEO, ISR, and storefront behavior.
- Keep every main form section expanded and ordered by editing priority.
- Desktop uses a sticky right rail; mobile uses a compact sticky section control, a Sheet outline, and a bottom action bar.
- Section navigation and manual scrolling stay synchronized through scrollspy behavior.
- Validation navigates to the first invalid field, activates the relevant section and locale, focuses the field, and renders an inline error.
- Bilingual content uses Vietnamese/English tabs, with locale error/readiness state visible even when a tab is inactive.

## Scope

- `src/components/admin/catalog/product-form.tsx`
- Focused tests if a practical component boundary exists.

## Tasks

1. Replace collapsible smart sections with always-visible semantic form sections and active-section styling.
2. Add click navigation and IntersectionObserver-based scrollspy with reduced-motion support.
3. Add locale tabs for content and SEO, including automatic locale switching for validation errors.
4. Add client validation based on the existing product schema, field-level error state, focus management, and section error counts.
5. Replace the duplicate sticky command bar with a compact desktop control rail and responsive mobile controls.
6. Keep workflow links and readiness metadata concise so the desktop rail fits normal laptop viewports.
7. Verify formatting, typecheck, lint, and responsive behavior in a real browser where authentication permits.

## Done Criteria

- Desktop editor has stable two-column layout and a right rail that stays within the viewport.
- Mobile has no forced two-column layout or horizontal overflow at 375px.
- Clicking a section control scrolls to and highlights the matching section.
- Manual scrolling updates the active section control.
- Invalid submit navigates to, focuses, and highlights the first invalid field, including fields inside inactive locale tabs.
- Existing catalog save/publish behavior remains unchanged.
- Typecheck and lint pass.
