# 260708-ops19 - Catalog Media Operational Instrumentation

## Goal

Record sanitized operational failures for catalog media admin actions so image and PDF upload, association, update, primary/social selection, and removal failures are visible to admins/developers without exposing file paths, file names, alt text, raw database details, emails, tokens, or stack traces.

## Scope

- Add focused unit coverage for catalog media failure recording.
- Instrument `src/catalog/media-actions.ts` error branches with `recordOperationalFailure`.
- Keep validation outcomes as validation outcomes; do not record user/admin input mistakes as operational failures.
- Preserve existing action return contracts and cache invalidation behavior.
