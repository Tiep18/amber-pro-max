# Replace admin product editor tabs with smart sections and taxonomy multi-select

## Goal

Replace the tabbed admin product editor with a more practical progressive-disclosure form for frequent product-entry work. Taxonomy must become compact multi-select with chips instead of long checkbox lists.

## Scope

- `src/components/admin/catalog/product-form.tsx`

## Tasks

1. Replace horizontal tabs with smart collapsible sections.
   - Keep section headers visible with completion/status cues.
   - Allow multiple sections to be open.
   - Avoid tabs with very little content.

2. Redesign taxonomy entry.
   - Categories, techniques, and tags use searchable multi-select controls.
   - Selected values render as compact chips with remove actions.
   - Collections use the same selection model plus display-order rows for selected collections.

3. Improve pricing and SEO grouping.
   - Keep content fields separate from SEO helper fields.
   - Render market offers as a compact matrix.

4. Preserve behavior.
   - Do not change server actions, schemas, publish checks, cache invalidation, or saved payload shape.

5. Verify.
   - Run `npm run typecheck`.
   - Run `npm run lint`.

## Done Criteria

- The form is no longer organized as horizontal tabs.
- Taxonomy entry is compact and searchable.
- Save/publish behavior remains unchanged.
- Typecheck and lint pass.
