# Account Customer Operational Instrumentation

## Goal

Record sanitized operational failures for customer account mutations that currently return generic errors without durable diagnostic context.

## Scope

- Wishlist add/remove persistence failures and unexpected persistence results.
- Saved shipping address save/delete/default RPC failures and unexpected RPC results.
- Keep customer PII out of operational facts: no raw recipient names, phone numbers, address lines, postal codes, emails, or tokens.

## Plan

1. Add failing unit tests proving wishlist and address failures call the operational recorder with safe facts.
2. Instrument the account actions using the existing `recordOperationalFailure` wrapper.
3. Run targeted account tests, typecheck, lint, then update GSD summary/state and commit only scoped files.

## Verification

- `npm run test:unit -- tests/unit/account/wishlist.test.ts tests/unit/account/addresses.test.ts`
- `npm run typecheck`
- `npm run lint`
