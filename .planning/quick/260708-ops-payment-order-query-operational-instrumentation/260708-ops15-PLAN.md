# Payment Order Query Operational Instrumentation

## Goal

Record sanitized operational failures for customer and admin order payment projection queries.

## Scope

- Customer authorized order payment lookup RPC failures and malformed found results.
- Admin order queue projection failures.
- Admin order detail lookup/timeline failures and malformed detail rows.
- Keep guest secret hashes, customer emails, shipping addresses, phone numbers, and raw database details out of operational facts.

## Plan

1. Add failing unit tests for recorder calls on payment/order query failures.
2. Instrument query failures with `recordOperationalFailure`.
3. Run targeted payments tests, typecheck, lint, update GSD summary/state, and commit scoped files only.
