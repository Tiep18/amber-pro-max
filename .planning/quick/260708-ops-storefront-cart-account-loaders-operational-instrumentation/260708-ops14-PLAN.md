# Storefront Cart And Account Loaders Operational Instrumentation

## Goal

Record sanitized operational failures for storefront/cart refresh and account load helpers that currently return generic errors without durable diagnostics.

## Scope

- Cart quote refresh action failures.
- Customer wishlist load failures.
- Customer shipping address load failures.
- Keep customer owner IDs, raw addresses, phone numbers, emails, tokens, and raw database details out of operational facts.

## Plan

1. Add failing unit tests for recorder calls in cart quote refresh, wishlist load, and address load failures.
2. Instrument the three paths with `recordOperationalFailure`.
3. Run targeted tests, typecheck, lint, then update GSD summary/state and commit scoped files only.

## Verification

- `npm run test:unit -- tests/unit/cart/actions.test.ts tests/unit/account/wishlist.test.ts tests/unit/account/addresses.test.ts`
- `npm run typecheck`
- `npm run lint`
