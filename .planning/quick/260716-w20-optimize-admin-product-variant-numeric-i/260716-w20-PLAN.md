# Quick Task 260716-w20: Optimize admin product variant numeric inputs

## Goal

Make every numeric field in the admin variants workspace fast and forgiving for day-to-day catalog work, especially on mobile, without changing inventory ownership, pricing rules, server actions, database schemas, or the atomic aggregate-save boundary. Reuse the existing `Input`, `Button`, design tokens, and `lucide-react`; add no runtime dependency.

## Scope guardrails

- In scope: product stock, variant quantity on hand, variant display order, and custom VND/USD override inputs in `VariantEditor`.
- Preserve `src/app/admin/catalog/[productId]/variants/page.tsx`, `src/catalog/variant-actions.ts`, `src/catalog/variant-schemas.ts`, `src/catalog/variant-pricing.ts`, and database behavior unless a type-only import adjustment is strictly required. Do not introduce a migration or remote data change.
- Keep `inherit | custom | unavailable` serialization unchanged: inherit omits the override, custom submits `enabled=true` and a parsed integer `priceMinor`, and unavailable submits `enabled=false` with the existing database-compatible zero price.
- Keep one aggregate action call per variant save. Do not split inventory or pricing into additional mutations and do not copy, infer, allocate, or exchange-convert stock or price.

## Task 1 - Add exact numeric primitives and integrate all four field types

**Files**

- `src/catalog/variant-numeric.ts` (new)
- `src/components/admin/catalog/numeric-stepper.tsx` (new)
- `src/components/admin/catalog/variant-editor.tsx`

**Action**

- Add small, pure helpers for UI numeric text:
  - Parse a required nonnegative whole-number string for stock/order. Reject blanks, signs, fractions, exponent notation, junk, and values above `Number.MAX_SAFE_INTEGER`; normalize valid text only on blur/save.
  - Step a whole-number string by an integer delta and clamp at zero. An invalid/blank field must remain editable; a step action may deliberately recover it from the zero baseline, but typing itself must never coerce `''` to `0`.
  - Parse custom money without `parseFloat`, floating-point multiplication, or locale guessing. Accept VND as whole đồng with optional valid comma grouping. Accept USD as dollars with optional valid comma grouping and zero, one, or two dot-decimal digits. Construct USD cents from whole/fraction digit strings, validate the safe-integer bound, and return an integer minor-unit result.
  - Format persisted values back to editor text (`250000` VND -> `250,000`; `1800` USD minor -> `18.00`) and use the existing currency-aware display contract for the effective preview. Invalid values return field-specific messages instead of a fallback zero.
- Build `NumericStepper` from the existing `Input` and `Button` with `Minus`/`Plus` icons. The compound control must have an explicit external `<label htmlFor>`, input `role="spinbutton"`, `inputMode="numeric"`, `aria-valuemin="0"`, `aria-valuenow` only when valid, `aria-invalid`/`aria-describedby`, ArrowUp/ArrowDown support, and labelled buttons such as `Decrease Quantity on hand by 1` and `Increase Quantity on hand by 1`. Disable decrement at zero, keep every target at least 44px, and avoid nesting buttons inside a `<label>`.
- Replace the immediate `Number(...) || 0` coercion for product stock, variant stock, display order, and custom price with UI-only text state. Initialize text from server-owned numbers, validate as the admin types/blurs, normalize valid values on blur, and build the immutable numeric domain snapshot only at product-inventory save or variant aggregate save.
- Include numeric validity in Save disabled state and local field errors. Preserve unsaved/discard behavior when a field contains blank or invalid text; canonical dirty comparison should compare parsed numeric intent when valid so formatting-only normalization does not manufacture a change, while invalid raw text still counts as an unsaved edit.
- Use the stepper for product stock, variant stock, and display order. Keep `-1/+1` as the primary semantic controls; add restrained stock-only quick actions such as `+5` and `+10` if they fit without crowding, with explicit accessible names and the same safe boundary logic. Do not add quick actions to display order or money.
- Rework each custom market field as a human-unit text input with a stable currency adornment and copy: VND is labelled in đồng and shows `VND`/`đ`; USD is labelled in dollars and shows `$`/`USD`. Preserve the last valid custom text while toggling market modes in the current draft, but serialize only the active mode. Show a formatted effective result and source beside the control.
- Keep the controls in compact responsive grids/flex rows that collapse cleanly at 375px, reserve error/help space without clipping, and use tabular figures. Do not change the page query, server actions, schemas, inventory ownership, pricing fallback, operation-token handling, or shipping section.

**Verify**

**Done when**

- Stock/order steppers have correct labels, keyboard behavior, boundary state, and mobile-safe layout; custom VND/USD entry is human-readable and converts exactly to the unchanged minor-unit payload.
