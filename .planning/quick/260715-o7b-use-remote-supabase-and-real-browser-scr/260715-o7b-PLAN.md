## Task 1 - Flatten localized sections and remove blank validation rows

**Files**

- `src/components/admin/catalog/product-form.tsx`

**Action**

- Change `FieldError` back to a truly conditional renderer: return no wrapper when `fieldErrors[path]` is absent; when present, render the same generated error ID on the same `p[role="alert"]`, with the exact message and destructive text styling. Preserve natural line wrapping and growth; do not reserve global empty height, absolutely position the message, clip it, or weaken `aria-invalid`/conditional `aria-describedby`.
- Replace `localizedPanelClassName`'s rounded border, translucent fill and nested padding with a flat field-group treatment inside `EditorFormSection`: a light top rule or heading divider is allowed, but there must be no second card silhouette around Content or SEO. Keep the existing tabpanel roles/IDs/`aria-labelledby`, locale state, paths, values and update handlers.
- Reduce mobile-only section/content inset, for example `p-3 sm:p-4`, and remove redundant inner horizontal padding so the 375px field width materially increases. Keep the outer section card, active edge, heading semantics and scroll-margin class unchanged.
- Keep locale tabs, labels and helper actions at least 44px. Redesign SEO actions as an explicit compact grid: three equal cells in one row when space permits and a deterministic two-row arrangement at 375px, with every existing button label and handler unchanged.
- Preserve top-aligned validation stability by keeping complex rows `items-start`; errors may increase their own field/section height but must not recenter or overlap sibling controls. The top result Alert remains naturally inserted and is not hidden.

**Verify**

- `npx.cmd eslint src/components/admin/catalog/product-form.tsx`
- `npm.cmd run typecheck`
- `git diff --check`
- Diff audit: validation messages, `fieldDomId`, every field path, `role="alert"`, `aria-invalid`, conditional `aria-describedby`, locale routing and focus/navigation handlers are byte-for-byte unchanged outside presentation markup.

**Done**

- Content and SEO have one outer section surface, no blank error row after valid fields, wider mobile inputs, and a predictable SEO action grid; the invalid JSON message remains visible, associated and focusable through the existing flow.
