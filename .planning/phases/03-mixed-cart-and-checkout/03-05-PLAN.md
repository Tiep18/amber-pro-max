<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Unavailable physical line to exception request | Shopper can request an exception but must not reserve stock or force availability. |
| Admin exception action to scoped grant | Approval can permit one exceptional checkout path and must be tightly scoped. |
| Grant token to checkout submit | Token is bearer-like input and must not bypass fresh validation or become reusable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05-01 | Spoofing | grant token | mitigate | 256-bit opaque token, hash at rest, expiry, generic invalid responses. |
| T-03-05-02 | Elevation of Privilege | exception scope | mitigate | Exact product/variant/country scope, admin-only approval, fresh validation. |
| T-03-05-03 | Tampering | grant reuse | mitigate | Atomic single-use consume in submit transaction and concurrency test. |
| T-03-05-04 | Information Disclosure | request/admin UI | mitigate | Mask email in list, no raw token logs, RLS owner/admin checks. |
| T-03-05-05 | Scope Creep | payment/fulfillment | mitigate | Security boundary test blocks PayPal/VietQR capture, entitlement, and fulfillment UI. |
</threat_model>
