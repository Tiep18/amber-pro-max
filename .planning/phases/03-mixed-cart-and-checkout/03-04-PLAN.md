## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Checkout form to submit RPC | Contact, destination, code, quote version, and idempotency are user-controlled. |
| Submit RPC to inventory/discount/order tables | One transaction must prevent oversell, overuse, and partial writes. |
| Guest access to pending order | Guest order visibility depends on scoped secret handling. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering | submit payload | mitigate | Full server recalculation, quote hash/version check, Zod validation. |
| T-03-04-02 | Tampering | inventory | mitigate | Stable row locks, active reservation subtraction, concurrency tests. |
| T-03-04-03 | Tampering | discount usage | mitigate | Lock usage rows and create pending redemption in same transaction. |
| T-03-04-04 | Information Disclosure | guest/account orders | mitigate | Hashed guest secret, owner RLS, generic errors, no raw token logs. |
| T-03-04-05 | Repudiation | duplicate submit | mitigate | Idempotency key returns same order and prevents duplicate reservations. |
| T-03-04-06 | Elevation of Privilege | definer RPC | mitigate | Fixed search_path, explicit grants, RLS-aware tests, no broad public execution. |
</threat_model>
