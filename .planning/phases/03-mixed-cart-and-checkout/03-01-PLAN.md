## Phase Goal

**As a** guest or signed-in shopper, **I want to** create a valid mixed order with server-calculated totals, shipping, discounts, destination validation, and inventory reservation, **so that** I can confidently continue to payment without stale prices, invalid stock, or hidden market changes.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser storage to cart quote | User-controlled localStorage can be edited or malformed. |
| Product route to add-to-cart | Shopper controls variant and quantity selection. |
| Quote response to UI | Server-calculated commercial data must be distinguished from browser intent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | guest cart | mitigate | Store only intent fields, validate with Zod, ignore commercial fields, test payload tampering. |
| T-03-01-02 | Information Disclosure | localStorage | mitigate | Never persist email, address, discount validity, payment data, quote hash, price, title, or stock. |
| T-03-01-03 | Tampering | quote | mitigate | Recalculate all line data from catalog queries and expose stale/change metadata. |
| T-03-01-04 | Accessibility | cart UI | mitigate | Keyboard sheet, aria-live count, text labels for product type and status, no color-only signals. |
</threat_model>
