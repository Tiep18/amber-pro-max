<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin discount form to database | Discount rules alter payable totals and usage limits. |
| Customer code input to quote | Shopper controls discount code and can replay or mutate requests. |
| Discount allocation to order snapshot | Allocations become commercial history in Plan 03-04. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Tampering | discount code | mitigate | Server-only validation, ignore browser discount state, safe failure messages. |
| T-03-03-02 | Tampering | money allocation | mitigate | Integer minor units, deterministic rounding tests, no floats. |
| T-03-03-03 | Elevation of Privilege | admin CRUD | mitigate | `requireAdmin()`, RLS, grants, no privileged UI for non-admin. |
| T-03-03-04 | Repudiation | usage accounting | mitigate | Prepare pending redemption records for Plan 03-04 transaction and idempotency. |
</threat_model>
