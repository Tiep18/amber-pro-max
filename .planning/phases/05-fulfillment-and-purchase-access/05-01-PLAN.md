**As a** paid digital or mixed-order customer, **I want to** securely access purchased PDFs and see separate fulfillment progress, **so that** I can receive each eligible part of my order only after payment is confirmed.

<objective>

Purpose: Phase 4 opens the paid gate; this plan makes that gate create durable digital access without making PDFs public or conflating digital readiness with physical fulfillment.
Output: Supabase fulfillment schema/RLS, paid-transition entitlement hook, typed fulfillment services, entitlement-checked download route, customer download panel, and focused entitlement/download/security tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| payment transition -> fulfillment schema | Paid state crosses into entitlement/outbox creation inside the database transaction. |
| customer/guest -> download route | Untrusted request attempts to obtain a private PDF signed URL. |
| app server -> Supabase Storage | Server creates short-lived signed URLs only after entitlement authorization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | `apply_payment_transition` fulfillment hook | mitigate | Unique entitlement/outbox constraints and paid-branch-only helper called under existing payment locks. |
| T-05-01-02 | Information Disclosure | `/api/downloads` | mitigate | Non-enumerating authorization, private bucket, no raw object paths in UI, signed URL only after entitlement validation. |
| T-05-01-03 | Spoofing | guest/download tokens | mitigate | Store SHA-256 hashes only, scope tokens to order/email/entitlement/purpose, enforce 24-hour expiry per D-03/D-07. |
| T-05-01-04 | Elevation of Privilege | entitlement reads | mitigate | RLS owner/admin policies plus server-side guest hash/token checks; no authenticated-only policy without owner predicate. |
| T-05-01-SC | Tampering | package installs | accept | No package installs in this plan. |
</threat_model>
