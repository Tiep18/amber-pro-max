## Phase Goal

**As a** mixed-order customer and store admin, **I want to** track physical fulfillment separately from PDF access, **so that** shipping progress is transparent without changing digital entitlement rules.

<objective>
Implement manual physical fulfillment, tracking, shipping notifications, and mixed-order customer status.

Purpose: Digital access can be complete before handmade goods ship. This plan adds the seller's manual physical workflow and customer tracking surface without collapsing the separate fulfillment state machines.
Output: Physical fulfillment transition contracts, admin tracking form, shipping email enqueue, localized customer tracking panel, mixed-order status updates, and physical fulfillment tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| admin browser -> physical fulfillment mutation | Privileged status/tracking updates enter order fulfillment state. |
| fulfillment mutation -> email outbox | Shipped status creates a customer-visible transactional notification. |
| customer order page -> tracking display | Customer sees shipping facts from server-authorized projections only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04-01 | Tampering | physical status transition | mitigate | Validate allowed state flow, load current state before mutation, reject stale/backwards transitions, and audit every change. |
| T-05-04-02 | Elevation of Privilege | physical fulfillment action/RPC | mitigate | `requireAdmin`, explicit database admin checks for any privileged SQL wrapper, narrow grants, and RLS/schema tests. |
| T-05-04-03 | Information Disclosure | tracking and admin notes | mitigate | Customer projection includes only carrier/tracking/customer-safe tracking URL/status, never admin note or internal event IDs. |
| T-05-04-04 | Repudiation | shipped email enqueue | mitigate | Transition event, outbox idempotency key, and email attempt records link shipped action to admin actor/order. |
| T-05-04-SC | Tampering | package installs | accept | No package installs in this plan. |
</threat_model>
