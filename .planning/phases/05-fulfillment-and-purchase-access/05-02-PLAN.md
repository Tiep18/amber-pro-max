## Phase Goal

**As a** paid digital or mixed-order customer, **I want to** receive secure transactional access and shipping emails, **so that** I can reopen purchases without email provider failures corrupting fulfillment.

<objective>
Build the transactional email outbox, localized email rendering, retry worker, and admin retry/resend operations.

Purpose: Fulfillment cannot depend on inline provider calls. This plan turns the durable outbox from 05-01 into a safe Resend-backed workflow with admin visibility.
Output: Resend dependency/config gate, transactional email renderer, outbox worker route, retry/backoff logic, admin failed-email queue, resend actions, and tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| database outbox -> email provider | Durable internal email intent becomes an external provider call. |
| admin browser -> retry/resend action | Privileged user can trigger new email work and entitlement-token issuance. |
| cron/worker -> route handler | Scheduled worker can process due rows only with a server secret. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | outbox worker | mitigate | Atomic claim, status transition constraints, idempotency keys, and attempt records. |
| T-05-02-02 | Information Disclosure | email templates/admin queue | mitigate | App links only, masked recipient in queue, sanitized errors, no raw tokens/signed URLs/object paths. |
| T-05-02-03 | Elevation of Privilege | retry/resend actions | mitigate | `requireAdmin`, stale row checks, audit rows, and no direct browser table mutations. |
| T-05-02-04 | Spoofing | worker route | mitigate | Bearer/shared secret, body limits, and typed unconfigured state when secret missing. |
| T-05-02-SC | Tampering | npm install `resend` | mitigate | Blocking human package legitimacy checkpoint before install because RESEARCH.md verdict is `SUS`. |
</threat_model>
