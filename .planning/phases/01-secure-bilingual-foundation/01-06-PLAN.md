<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form -> auth server actions | Untrusted form values and redirect targets enter server actions. |
| Supabase Auth email -> localized callback | Confirmation and recovery links return through a public callback route. |
| provider error -> customer UI | Provider failures must be mapped to safe localized copy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-01 | Information Disclosure | auth error rendering | mitigate | Map provider errors to stable localized messages and never render raw Supabase strings. |
| T-01-06-02 | Information Disclosure | reset request UI | mitigate | Always show generic success after reset request submission. |
| T-01-06-03 | Tampering | auth form submissions | mitigate | Use Plan 01-05 Zod schemas and server actions for authoritative validation. |
| T-01-06-04 | Spoofing | localized callback links | mitigate | E2E tests verify localized callback handling and safe redirect preservation. |
</threat_model>
