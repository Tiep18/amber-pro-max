<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server environment -> client modules | Public and secret configuration must remain separated by import boundary. |
| browser request -> composed proxy | Untrusted cookies and paths flow through locale and auth cookie refresh. |
| browser form -> server action | Email, password, redirect target, and locale are untrusted input. |
| Supabase Auth -> app callback | Auth codes and recovery sessions must be exchanged and validated server-side. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05-01 | Information Disclosure | Supabase client modules | mitigate | Split browser/server/proxy clients and expose only publishable values to browser-importable code per SEC-02. |
| T-01-05-02 | Spoofing | Supabase SSR session refresh | mitigate | Use official SSR cookie helpers and preserve `Set-Cookie` on the composed proxy response. |
| T-01-05-03 | Information Disclosure | password reset request | mitigate | Return generic success for known and unknown email addresses. |
| T-01-05-04 | Spoofing | post-auth redirects | mitigate | Validate same-origin localized relative `next` paths only; reject external, malformed, protocol-relative, unsupported-locale, and unprefixed targets. |
| T-01-05-05 | Elevation of Privilege | auth service boundary | mitigate | Do not authorize from `user_metadata`; protected admin checks are database-backed in Plan 01-07. |
</threat_model>
