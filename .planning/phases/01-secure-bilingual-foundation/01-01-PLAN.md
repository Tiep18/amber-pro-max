<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry -> local install | Package metadata and tarballs enter the development environment. |
| package scripts -> developer machine | Lifecycle scripts can execute during install. |
| environment contract -> browser bundle | `NEXT_PUBLIC_*` names become browser-visible configuration. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-SC | Tampering | npm installs | mitigate | Blocking human package legitimacy checkpoint before install; verify official owner/repository/version and inspect lifecycle scripts. |
| T-01-01-01 | Information Disclosure | `.env.example` | mitigate | Document only publishable Supabase and site URL variables; reject service-role or secret names in public config per SEC-02. |
| T-01-01-02 | Tampering | shadcn source generation | mitigate | Use official shadcn registry/source only and record generated source in later UI plan. |
</threat_model>
