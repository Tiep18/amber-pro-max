<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| source/build output -> browser | Static assets and client modules must not contain privileged credentials or server-only modules. |
| CI runner -> local Supabase | Automated verification must use reproducible local database state, not privileged shortcuts. |
| operator dashboard -> hosted app | Hosted Supabase/Vercel settings affect redirect safety, SMTP, and deployability. |
| human visual review -> release confidence | Some responsive, translation-fit, and perceived-boundary checks require human confirmation after automation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08-01 | Information Disclosure | `.next/static` and client source | mitigate | `scripts/check-secrets.mjs` scans source and build output for secret/service-role indicators and privileged imports. |
| T-01-08-02 | Elevation of Privilege | CI database tests | mitigate | CI runs local `supabase db reset`, lint, pgTAP, and generated type checks before E2E. |
| T-01-08-03 | Spoofing | hosted Auth redirect configuration | mitigate | Operator checkpoint verifies exact Site URL and redirect allowlist for localhost, preview, callback, and localized destinations. |
| T-01-08-04 | Information Disclosure | environment badges and diagnostics | mitigate | E2E checks ensure badges never display project IDs, tokens, service keys, or raw provider payloads. |
| T-01-08-05 | Denial of Service | Auth email in production | transfer | Supabase SMTP configuration is operator-owned; checkpoint blocks production acceptance until custom SMTP status is confirmed. |
</threat_model>
