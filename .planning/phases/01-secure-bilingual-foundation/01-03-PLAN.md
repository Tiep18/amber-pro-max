<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localized messages -> rendered UI | Translated copy becomes visible to customers and must not expose raw keys or diagnostics. |
| user interaction -> locale switcher | Query parameters and current paths are reused during language switching. |
| UI visibility -> authorization expectations | Navigation can guide users but must never be treated as an authorization control. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | localized shell messages | mitigate | Render approved localized copy only; missing keys fail tests and do not expose raw internal keys. |
| T-01-03-02 | Spoofing | `src/components/locale-switcher.tsx` | mitigate | Use typed routing helpers and safe query allowlist so language switching cannot become an open redirect. |
| T-01-03-03 | Elevation of Privilege | public navigation | accept | Public navigation is not an authorization boundary; protected account/admin checks are implemented server-side in later plans. |
</threat_model>
