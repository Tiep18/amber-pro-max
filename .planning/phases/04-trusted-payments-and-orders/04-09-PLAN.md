<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| admin route -> privileged evidence | Admin pages expose sensitive operational facts. |
| admin form -> payment transition | Evidence and decision fields are untrusted and can be replayed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-23 | Information disclosure | admin evidence | mitigate | Server admin auth, redacted provider facts and no raw payload/signature display. |
| T-04-24 | Tampering / replay | VietQR form | mitigate | Server Action validation, idempotency key, stale-state handling and DB-side checks. |
| T-04-25 | Repudiation | admin decisions | mitigate | Timeline/audit includes actor, timestamp, reason, evidence and transition result. |
| T-04-SC | Tampering | package supply chain | accept | Use existing shadcn/internal components and Lucide icons only. |
</threat_model>
