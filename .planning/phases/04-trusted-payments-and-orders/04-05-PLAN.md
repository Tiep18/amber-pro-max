<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> PayPal route | Browser can request a provider order but cannot control amount, receiver, or shipping facts. |
| PayPal provider -> local capture route | Provider response must be reconciled before local paid state changes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | Tampering | PayPal create payload | mitigate | Derive amount, currency, merchant, order number, and shipping context from server projections only. |
| T-04-14 | Spoofing | PayPal capture result | mitigate | Reconcile provider order/capture/merchant/amount/currency before transition. |
| T-04-15 | Information Disclosure | route logs/errors | mitigate | Log sanitized facts only, no credentials, raw payloads, tokens, or customer address in public errors. |
| T-04-SC | Tampering | package installs | accept | No PayPal SDK package; direct REST client uses existing fetch. |
</threat_model>
