<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser checkout -> Server Action/RPC | Customer-submitted contact, address, cart intent, and payment intent cross into trusted order creation. |
| provider/admin/cron -> apply_payment_transition | External or privileged payment facts can alter money, stock, and paid gate state. |
| customer/admin order views -> projections | Sensitive order, address, provider, and audit evidence must be filtered by role. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | address submit payload | mitigate | Tests require server validation, normalized ISO country, quote revalidation, and immutable DB snapshot before payable order creation. |
| T-04-02 | Information Disclosure | order views | mitigate | Tests require owner/guest/admin scoped projections and no cross-customer address leakage. |
| T-04-03 | Tampering | payment transitions | mitigate | Preserve shared idempotent transition contracts for PayPal, VietQR, expiry, and inventory effects. |
| T-04-SC | Tampering | package installs | accept | No npm, pip, or cargo installs are planned; country selection must use owned code and platform APIs. |
</threat_model>
