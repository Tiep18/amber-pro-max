<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| browser/session -> order query | Untrusted order number and identity can target another customer's order. |
| server -> privileged Supabase client | Secret key bypasses RLS and must remain server-only. |
| checkout result -> guest browser | Raw guest token must not enter URL, storage or logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-05 | Information disclosure | order projections | mitigate | Owner/hash scope, generic denial and separate customer/admin projections. |
| T-04-06 | Information disclosure | env/guest tokens | mitigate | `server-only`, non-public env names, scoped HttpOnly cookie and static scan. |
| T-04-07 | Elevation of privilege | admin query/action boundary | mitigate | `requireAdmin()` plus database RLS/private admin checks. |
| T-04-SC | Tampering | package supply chain | accept | No package changes beyond adding the existing Node security test to scripts. |
</threat_model>
