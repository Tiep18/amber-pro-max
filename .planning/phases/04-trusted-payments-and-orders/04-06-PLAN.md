## Trust Boundaries

| Boundary | Description |
|---|---|
| internet -> webhook route | Anyone can forge, replay or oversize requests. |
| verified event -> payment transition | False acceptance would affect money, stock and gate state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-04-14 | Spoofing | PayPal webhook | mitigate | Official raw-body signature verification with configured webhook ID. |
| T-04-15 | Tampering | replay/out-of-order delivery | mitigate | Unique provider event ID, delivery counter and monotonic shared transition. |
| T-04-16 | Information disclosure / DoS | webhook payload | mitigate | Body cap, digest plus sanitized facts, bounded calls and no raw logs/storage. |
| T-04-SC | Tampering | package supply chain | accept | Existing Web `fetch`; no package install. |
</threat_model>
