<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Overlapping async responses -> checkout state | Older responses are untrusted once a newer destination request starts. |
| Saved account address -> quote intent | Stored convenience data must be normalized and re-quoted, not accepted as price evidence. |
| Browser quote intent -> server action | Amounts/configuration IDs must not cross the validated boundary. |
