<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/server action -> quote RPC | Cart/destination input is untrusted and may contain forged shipping evidence. |
| Accepted browser quote -> submit transaction | Hashes and quote JSON are advisory until every commercial/shipping fact is re-resolved. |
| Submit transaction -> immutable order evidence | Configuration can change concurrently; accepted evidence must be written atomically and never edited. |

## ASVS L1 / STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | Tampering | Quote/submit boundary | mitigate | Validate bounded intent, ignore browser fees/IDs, invoke authoritative resolver inside submit, and create no rows on mismatch. |
| T-08-06 | Tampering / Repudiation | Allocation snapshot | mitigate | Insert complete evidence in the order transaction, enforce sum equality/FKs/checks, and block update/delete with trigger tests. |
| T-08-07 | Tampering / Information Disclosure | Stale/material-change response | mitigate | Hash canonical allocation dimensions, return only allowlisted stale codes/dimensions, and redact SQL/internal facts. |
| T-08-SC | Tampering | Dependency surface | accept | No new package or dependency is introduced. |

ASVS L1 gate: server-side validation, authorization, transaction integrity, fail-closed pricing, and sanitized errors must pass. Any HIGH/CRITICAL finding or failed T-08-05/T-08-06/T-08-07 mitigation blocks completion.
</threat_model>
