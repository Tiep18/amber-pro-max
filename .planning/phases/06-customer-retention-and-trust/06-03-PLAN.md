<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Visitor to subscribe action | Anonymous email, locale, market, and request metadata cross into consent records. |
| Email link to unsubscribe handler | Bearer token from email crosses into unauthenticated state mutation. |
| Consent evidence to admin UI | Sensitive request metadata is summarized for admin inspection. |
| Admin newsletter page to consent state | Admin can inspect but must not mutate subscription consent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Spoofing | Subscribe action | mitigate | Zod email/locale/market validation, explicit submit control, generic safe result, and account-independent email identity per D-13. |
| T-06-03-02 | Tampering | Unsubscribe token flow | mitigate | High-entropy raw token, SHA-256 hash-only storage, expiry/consumed metadata, generic localized result states, and no sign-in requirement per D-14. |
| T-06-03-03 | Information Disclosure | Consent event metadata and admin list | mitigate | Store hashed/redacted IP/user-agent metadata, never raw request dumps or raw token values, and add static security checks per D-16. |
| T-06-03-04 | Elevation of Privilege | Admin subscriber page | mitigate | `requireAdmin` protects reads; no admin subscribe/unsubscribe action or RPC exists per D-15. |
| T-06-03-SC | Tampering | npm package surface | accept | Plan installs no packages and reuses the existing Resend/outbox stack. |
</threat_model>
