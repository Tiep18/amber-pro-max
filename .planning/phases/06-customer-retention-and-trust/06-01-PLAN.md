<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to address/wishlist actions | Untrusted form inputs cross into customer-owned mutations. |
| Account data to checkout | Mutable saved-address convenience data is copied into checkout state but must not become order evidence. |
| Wishlist to catalog projections | Stored product IDs are hydrated into current market-aware facts. |
| Guest heart to auth redirect | Anonymous intent crosses into localized sign-in return handling. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Elevation of Privilege | `customer_shipping_addresses`, `wishlist_items`, server actions | mitigate | Use `requireUser`, server-owned `user.id`, RLS owner predicates using `(select auth.uid())`, and cross-user pgTAP tests. |
| T-06-01-02 | Tampering | Saved-address default and checkout shipping evidence | mitigate | Enforce one-default partial unique index, omit order FK to saved addresses, and test checkout order shipping snapshot immutability per D-01 through D-04. |
| T-06-01-03 | Information Disclosure | Wishlist current catalog facts | mitigate | Store only product references and hydrate through existing public catalog projections; security test rejects wishlist price/title/stock snapshot columns per D-05. |
| T-06-01-04 | Spoofing | Guest wishlist heart return flow | mitigate | Anonymous heart controls route to localized sign-in with validated relative `next`; no guest wishlist table or merge path per D-08. |
| T-06-01-SC | Tampering | npm package surface | accept | Phase 6 Plan 01 installs no packages; Package Legitimacy Audit says no new package. |
</threat_model>
