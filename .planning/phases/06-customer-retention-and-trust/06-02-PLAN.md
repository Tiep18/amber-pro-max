<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to review actions | Untrusted review content, rating, and product IDs cross into purchase eligibility checks. |
| Orders to review eligibility | Paid order-line evidence decides whether a customer can review. |
| Admin moderation to public product page | Admin state changes decide which review content becomes public. |
| Review identity to public UI | Customer identity must be transformed before public rendering. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Spoofing | `submitProductReviewAction`, review eligibility SQL/RPC | mitigate | Derive verified purchase from paid `checkout_order_lines` and paid gate/order status; reject client `verifiedPurchase` flags. |
| T-06-02-02 | Tampering | Customer review update | mitigate | Unique customer/product review row; update existing row and set status to `pending` on content edits per D-09 and D-10. |
| T-06-02-03 | Information Disclosure | Public review components/projections | mitigate | Approved-only query/projection, masked display names, and static checks rejecting full emails per D-11. |
| T-06-02-04 | Elevation of Privilege | Admin moderation/reply actions | mitigate | `requireAdmin`, `private.is_admin()`, expected version checks, direct grant restrictions, and pgTAP function privilege tests. |
| T-06-02-SC | Tampering | npm package surface | accept | Plan installs no packages; use existing React, Next.js, Zod, Supabase, and lucide dependencies. |
</threat_model>
