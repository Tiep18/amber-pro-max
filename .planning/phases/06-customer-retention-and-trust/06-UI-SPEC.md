# Phase 6 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | `new-york`, Tailwind v4, CSS variables, neutral base |
| Component library | Radix-backed shadcn primitives; installed: `alert`, `button`, `card`, `separator`, `sheet`, `skeleton` |
| Icon library | `lucide-react`; use `Heart`, `Star`, `ShieldCheck`, `Mail`, `Trash2`, `Pencil`, `CheckCircle2`, `CircleX`, `Clock3` as needed |
| Font | Existing app font stack; no new font package in Phase 6 |

Source notes: detected from `components.json`, `npx shadcn info`, `src/app/globals.css`, and existing UI primitives. Phase 6 must preserve the warm storefront tokens and dense admin/account patterns already in the app.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, status-dot gaps, inline badge spacing |
| sm | 8px | Compact element spacing, form helper spacing, adjacent action gaps |
| md | 16px | Default form/control gaps, account section spacing, product card internal padding |
| lg | 24px | Card padding, form groups, review/admin section padding |
| xl | 32px | Page header to content gap, major account/admin list spacing |
| 2xl | 48px | Major public section breaks, product detail review section separation |
| 3xl | 64px | Page-level separation only; avoid inside admin queues |

Exceptions: interactive targets must be at least 44px high/wide even when the visual icon is 20px or 24px. Product images and wishlist cards must reserve stable aspect ratios before data loads. Use existing `--radius-control: 8px`; for new Phase 6 cards use `--radius-card` only at the outer repeated-item boundary, never nested cards.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label / metadata | 14px | 600 | 1.4 |
| Body / form text | 16px | 400 | 1.5 |
| Section heading | 20px | 600 | 1.3 |
| Page heading | 30px | 600 | 1.2 |

Use exactly these two weights for new Phase 6 UI: regular `400` and semibold `600`. Keep letter spacing at `0`. Wrap Vietnamese and English copy before truncating; only truncate product titles in existing catalog cards where the current component already does so. Prices, counts, review totals, and dates should use tabular figures when they sit in rows or columns.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fff9f2` / `var(--background)` | Page background for account, product, newsletter, and public unsubscribe pages |
| Secondary (30%) | `#ffffff` / `var(--surface)` plus `#f5ede3` / `var(--surface-muted)` | Cards, rows, status chips, muted admin panels, selected address/wishlist states |
| Accent (10%) | `#a6472d` / `var(--accent)` | Primary CTAs, product/detail heart selected state, focus rings, active filter chips, verified badge icon only when paired with text |
| Destructive | `#b42318` / `var(--destructive)` | Delete address, remove admin reply, hide review, destructive confirmations only |

Accent reserved for: primary submit buttons, selected wishlist heart fill/stroke, active account/admin filters, verified-purchase icon highlight, focused form controls, and hover/focus border on repeated clickable rows. Do not use accent for all links or dense admin metadata.

Status colors: use existing `--success`, `--warning`, and `--destructive` only with text labels or icons. Never communicate availability, moderation, subscription, or eligibility state by color alone.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | By surface: `Save address` / `Luu dia chi`; `Add to wishlist` / `Luu yeu thich`; `Submit review` / `Gui danh gia`; `Subscribe` / `Dang ky nhan tin`; admin moderation primary `Approve review` |
| Empty state heading | `Nothing saved yet` |
| Empty state body | Account addresses: `Save a shipping address to fill checkout faster next time.` Wishlist: `Save products from the shop and check current availability here.` Reviews: `No approved reviews yet.` Newsletter admin: `No subscribers match these filters.` |
| Error state | `We could not save your changes. Check the highlighted fields and try again.` For server eligibility failures: `This action is no longer available. Refresh the page to see the current status.` |
| Destructive confirmation | `Delete address`: `Delete this saved address? Past orders keep their original shipping details.` `Hide review`: `Hide this review from the storefront?` `Remove reply`: `Remove the public admin reply?` |

Copy rules:
- Public and account copy must be localized through `next-intl`; do not hardcode English in reusable components.
- Vietnamese labels must fit in the same controls as English at 375px width. Buttons wrap only as a last resort; prefer short verb+noun labels.
- Avoid visible instructional text about how hearts, filters, forms, or shortcuts work. Explain only consequences: unavailable market, pending moderation, immutable order snapshots, unsubscribe result.
- Public identity copy must use masked display such as `A***@example.com` or a shortened display name, never a full email address.

## Phase Surface Contracts

### Account Addresses

| Area | Contract |
|------|----------|
| Page shell | Reuse the existing account width rhythm: `max-w-[760px]` for single-column forms and up to `max-w-[1040px]` for list plus editor layouts. Use one top-level `Card`; repeated addresses are bordered rows or radio-like panels inside it, not nested cards. |
| Address row | Each saved address row has a stable min height of 72px desktop and 88px mobile, a visible label/name, recipient, country, and `Default` badge when applicable. Row actions are `Edit`, `Delete`, and `Use in checkout` where context allows. |
| Default state | One default address is indicated with a muted pill plus semibold text. Changing default gives inline success feedback without reordering the list unexpectedly. |
| Checkout selection | In checkout, selecting a saved address copies values into the existing shipping form and immediately triggers quote revalidation. If material changes exist, reuse the Phase 3/4 quote-diff confirmation pattern before continuing. |
| Edit/delete | Edit opens an inline form or sheet with visible labels and field errors. Delete requires confirmation copy from this spec. Deleting a default address leaves no hidden default until the user or server chooses the next default explicitly. |
| Accessibility | Address selection uses radio semantics or buttons with `aria-pressed`; action buttons have visible labels on desktop and accessible names on icon-only mobile layouts. |

### Wishlist

| Area | Contract |
|------|----------|
| Product card heart | Add a top-right heart control inside the image/product-card boundary without shifting product title, price, badge, or CTA. Hit area is 44px; icon is 20px or 24px. Selected state uses filled/stroked accent heart plus `aria-pressed=true`. |
| Product detail heart | Place near the product title/CTA group as a secondary action. The primary product CTA remains add/view/cart behavior; wishlist must not compete visually with buying actions. |
| Guest behavior | Guest heart routes to localized sign-in with return path to the product or wishlist context. Use a concise inline/auth redirect message; do not create a guest wishlist or merge UI. |
| Account wishlist | Render product-level saved items as responsive repeated rows/cards with stable image aspect ratio, current price, current market availability, stock/variant status, and remove action. |
| Unavailable state | Products unavailable in the active market stay visible with a warning-surface status pill and disabled checkout/action. Show the reason next to the product facts, not as a modal. |
| Removal | Removing from wishlist is immediate with non-blocking success feedback. The remove control must be at least 44px and labeled `Remove`. |

### Verified Reviews

| Area | Contract |
|------|----------|
| Rating model | Use a conventional 1-5 integer star rating. Stars are buttons/radio options with 44px hit areas, visible selected state, keyboard support, and text equivalent such as `4 of 5 stars`. |
| Review entry | Eligible signed-in customers see one review form per product. Existing review edits reuse the same form and clearly show `Pending moderation` after submit/edit. Ineligible users see no form; show only approved public reviews and a compact eligibility note where needed. |
| Public display | Product pages show approved reviews only. Each item includes rating, short title/body if present, masked identity, date, and a `Verified purchase` badge with `ShieldCheck`. Full customer email is never public. |
| Admin reply | One admin reply appears as an indented muted-surface block inside the review item, not as a nested card. Label it `Shop reply`; admin can edit or remove it from the protected admin surface. |
| Moderation admin | Use the existing admin queue style: one top-level card, compact filter row, bordered grid rows with status, product, masked/full operational customer email as permitted, rating, submitted date, and actions. Primary action is `Approve review`; secondary actions are `Hide review`, `Edit reply`, `Remove reply`. |
| State changes | Customer edits return the review to pending moderation and remove it from public display until approved. Admin actions must show loading/disabled states and inline error recovery. |

### Newsletter Consent

| Area | Contract |
|------|----------|
| Public subscribe | Use a compact form with visible `Email` label, locale/market captured from context, and one primary `Subscribe` button. The form can live in footer/public sections but must not become a marketing hero in this phase. |
| Consent copy | State that subscription is explicit and can be unsubscribed from email. Keep body copy to one short sentence per locale. No pre-checked consent boxes. |
| Success state | After subscribe, show `You're subscribed.` plus the submitted email in masked form if shown. Do not expose raw consent metadata. |
| Unsubscribe page | Token route is one-click and does not require sign-in. The localized result page has three states: success, already unsubscribed/expired generic success, and invalid token with recovery path to contact or resubscribe. Include a secondary `Subscribe again` action. |
| Admin subscribers | Protected admin page is inspect-only for v1. Use dense filters for status, locale, market, and search. List shows email for operational search, status, latest locale/market, latest consent source, latest consent time, and redacted/hash metadata indicators. No manual subscribe/unsubscribe buttons. |

### Admin And Account Density

| Area | Contract |
|------|----------|
| Admin layout | `max-w-[1040px]`, `px-4 sm:px-6`, `py-10`, compact row grids, no decorative imagery, no hero treatments. |
| Storefront/account layout | Polished but restrained: product imagery stays inspectable; account forms are calm and clear; newsletter/review surfaces do not use oversized marketing copy. |
| Mobile | At 375px, all rows stack to one column, action groups wrap, no horizontal scroll, and fixed/sticky action areas reserve space so content is not covered. |
| Loading | Use existing `skeleton` for lists and form sections that load longer than 300ms. Buttons disable while submitting and keep their size stable. |
| Empty/error | Empty states are text plus one relevant action only. Errors appear near the field or row that caused them, with a clear retry/edit path. |
