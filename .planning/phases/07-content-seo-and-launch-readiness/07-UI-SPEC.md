## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage | Source |
|-------|-------|-------|--------|
| xs | 4px | Icon gaps, segmented-control internal gaps, status dot spacing | Default |
| sm | 8px | Compact control gaps, badge padding, list row gaps | Existing Tailwind classes |
| md | 16px | Default form/grid/card content spacing, mobile page padding | Existing Tailwind classes |
| lg | 24px | Card padding, section grouping, admin panel gaps | Existing `Card` p-6 |
| xl | 32px | Page section breaks and footer/header vertical rhythm | Existing layout |
| 2xl | 48px | Major public content breaks | Default |
| 3xl | 64px | Page-level top/bottom spacing on public content pages | Default |

Exceptions:
- Interactive controls use minimum touch target `44px` (`min-h-11` or `min-w-11`) even when visual padding is smaller.
- Existing header heights remain `64px` mobile and `72px` desktop.
- Media uses stable aspect ratios: blog/social image `16/9`, product cards `4/3`.
- Existing project tokens use `--radius-control: 8px`, `--radius-card: 12px`, and `--radius-surface: 16px`; new compact operational rows should prefer `--radius-control` unless reusing the existing `Card` primitive.

## Typography

Use exactly these four new-UI text sizes for Phase 07 surfaces; inherited existing components may keep their current internal sizes.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Meta / Label | 14px | 600 | 1.4 | Form labels, queue labels, publish blockers, status text |
| Body | 16px | 400 | 1.5 | Prose, controls, descriptions, field help |
| Heading | 20px | 600 | 1.3 | Card titles, admin section headings, blog card titles |
| Page Title | 28px | 600 | 1.2 | Public blog/policy H1 and admin page H1 |

Rules:
- Use only weights `400` and `600`.
- Letter spacing remains `0`; do not use negative tracking.
- Blog/policy prose uses `16px / 1.5` minimum; line length should stay near `65-75ch`.
- Admin tables/lists may use `14px` metadata, but primary values and actions stay legible at `16px`.

## Color

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#fff9f2` (`--background`) | Page background and public content bands | `src/app/globals.css` |
| Secondary (30%) | `#ffffff` (`--surface`) and `#f5ede3` (`--surface-muted`) | Cards, nav, form fields, row backgrounds, empty states | `src/app/globals.css` |
| Accent (10%) | `#a6472d` (`--accent`), hover `#843823` | Primary CTA, active locale/market segment, focused border, selected filter, product/blog/policy link emphasis, actionable dashboard link hover | `src/app/globals.css` |
| Destructive | `#b42318` (`--destructive`) | Destructive actions only: unpublish, delete draft, discard unsaved content, clear launch evidence | `src/app/globals.css` |

Accent reserved for: primary CTA buttons, active segmented controls, focused outlines, selected filters, inline links that navigate to key content, item-count badges that require action, and hover/focus borders on actionable admin rows.

Semantic colors:
- Success uses `#2f6b4f` and `#e9f4ed` for saved/published/resolved states.
- Warning uses `#8a5a16` and `#fff4d6` for scheduled, incomplete, pending UAT, and launch-blocking warnings.
- Destructive is not used for ordinary validation hints unless the form submission failed or the action is destructive.

WCAG contract:
- Text and interactive states target WCAG AA contrast.
- Color is never the only signal; include text labels, icons, or status words.

## Copywriting Contract

Artifact language is English; implementation copy must be localized in Vietnamese and English through existing message patterns.

| Element | Copy |
|---------|------|
| Primary CTA | Blog editor: `Publish post`; scheduled state: `Schedule post`; policy editor: `Publish policy`; launch settings: `Save launch settings`; operations queue: `Mark resolved` |
| Empty state heading | Blog admin: `No blog posts yet`; policies: `No published policy yet`; operations: `No unresolved operational errors`; launch checklist: `No launch checks configured` |
| Empty state body | Blog admin: `Create the first bilingual post before launch.` Policies: `Publish the required bilingual policy before linking it in checkout.` Operations: `New sanitized errors will appear here when they need review.` Launch checklist: `Add required launch decisions before final verification.` |
| Error state | Blog/policy forms: `This content could not be saved. Review the highlighted fields and try again.` Operations queue: `Operational errors could not be loaded. Refresh the page or check server logs with sensitive data redacted.` |
| Destructive confirmation | Unpublish post: `Unpublish this post? It will be removed from public blog pages and sitemaps.` Unpublish policy: `Unpublish this policy? Launch readiness will fail until it is published again.` Delete draft: `Delete this draft? This cannot be undone.` Clear launch evidence: `Clear this launch evidence? The checklist item will become unresolved.` |

Interaction copy rules:
- CTA labels use verb plus noun.
- Validation messages state the exact missing field: `Vietnamese slug is required before publishing.`
- SEO blockers name the page and locale: `English social image is required before this post can be published.`
- Public-facing policy links use customer terms, not internal requirement names.

## Interaction States

Every new public/admin form and queue in Phase 07 must implement these states.

| State | Required Behavior |
|-------|-------------------|
| Loading | Disable submitting controls, preserve entered values, show `Skeleton` or inline pending label sized to the final content |
| Validation | Field-level message with `aria-describedby`; publish blockers grouped near submit controls; focus moves to the first invalid field after failed submit |
| Error | Use `Alert` destructive for form-level failure; do not reveal provider payloads, stack traces, signatures, secrets, tokens, or unnecessary PII |
| Empty | Use text plus one next action; no blank panels |
| Disabled | Disabled control explains the blocker in adjacent helper text; cursor and opacity match existing `Button` behavior |
| Success | Use `role="status"` and success semantic color; success copy states what changed and where to go next |
| Preview | Blog/policy preview is admin-only, locale-specific, and visibly marked as preview without adding it to public navigation or sitemap |
| Scheduled | Scheduled posts show date/time, timezone context, and public-visibility explanation; no background job status is shown because visibility is query-owned |

Accessibility rules:
- All controls have visible labels.
- Keyboard users can reach filters, locale tabs, preview actions, publish controls, and operation detail sheets.
- Mobile layout must have no horizontal scroll at `375px`.
- Locale, currency, market, and shipping country remain conceptually separate in copy and controls.
